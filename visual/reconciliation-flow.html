Nous Reconciliation Flow

observe → diff → act — the core control loop pattern

1
Trigger — Watch Event or Timer
The informer receives a WatchEvent{ADDED|MODIFIED|DELETED} from the API server's gRPC streaming endpoint, or a periodic timer fires.
// Watch stream client in informer stream, _ := client.WatchAgentDefinitions(ctx, namespace, resourceVersion) for { event, _ := stream.Recv() switch event.Type { case "ADDED", "MODIFIED", "DELETED": queue.Add(namespace + "/" + event.AgentDefinition.Metadata.Name) } }
2
WorkQueue — Rate-Limited, Deduplicating
The key is enqueued. Multiple adds of the same key are deduplicated. If the key is currently being processed, it's marked dirty and re-enqueued after Done().
// Key transitions queue.Add("default/researcher") // → queue: [researcher] queue.Add("default/researcher") // → no-op (already queued) key := queue.Get() // → queue: [], processing: {researcher} queue.Add("default/researcher") // → dirty: {researcher} queue.Done(key) // → queue: [researcher] (re-enqueued)
3
Observe — Read Desired and Actual State
Worker goroutine dequeues the key and reads both desired state (AgentDefinition spec) and actual state (existing AgentInstances).
// Desired state def, _ := client.GetAgentDefinition(ctx, namespace, name) desired := def.Spec.Scaling.DesiredInstances // e.g. 2 // Actual state instances, _ := client.ListAgentInstances(ctx, namespace, name) actual := len(instances) // e.g. 0
4
Diff — Compare Desired vs Actual
Three possible outcomes drive different actions:
desired > actual
Scale up. Create (desired − actual) new AgentInstances.
desired < actual
Scale down. Delete (actual − desired) excess instances (graceful).
desired == actual
No-op for scaling. Still update status and health conditions.
5
Act — Execute Scale-Up / Scale-Down
Create or delete AgentInstance records. All writes are fenced with the current leader election FenceToken — stale writes are rejected by DynamoDB.
// Scale up — create missing instances for i := 0; i < desired-actual; i++ { inst := buildAgentInstance(def, generateShortID()) client.CreateAgentInstance(ctx, inst) // TODO: needs proto RPC } // Scale down — delete excess instances (oldest first) for i := 0; i < actual-desired; i++ { client.DeleteAgentInstance(ctx, instances[i]) // TODO: needs proto RPC }
6
UpdateStatus — Reflect Actual State
Update the AgentDefinition's status subresource with current counts and conditions. Uses optimistic concurrency — if the definition changed during reconciliation, write fails and key is requeued.
def.Status.ReadyInstances = countReady(instances) def.Status.DesiredInstances = int32(desired) def.Status.UnavailableInstances = countUnavailable(instances) def.Status.Conditions = []Condition{ {Type: "Available", Status: boolToStr(readyCount >= desired)}, {Type: "Progressing", Status: boolToStr(readyCount < desired)}, } client.UpdateAgentDefinition(ctx, def)
7
Error Handling — Exponential Backoff
On failure, the key is re-enqueued with exponential backoff. On success, backoff is forgotten. A RequeueAfter schedules the next periodic health-check reconciliation.
1st error → 5ms
→
2nd → 10ms
→
3rd → 20ms
→
doubles each time
→
max 1000s
if err != nil { queue.AddRateLimited(key) // exponential backoff return } queue.Forget(key) // reset backoff counter queue.AddAfter(key, reconcileInterval) // periodic re-check
