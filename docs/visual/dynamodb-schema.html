Nous DynamoDB Schema

single-table design — nous-state — all resource types in one table

Primary Key Structure
Attribute Type Pattern Example Purpose
PK S NS#<namespace> NS#default Partition by namespace — all resources in a namespace are co-located
SK S TYPE#<name> AGENTDEF#researcher Sort key uniquely identifies the resource within its partition
GSI1PK S NS#<ns>#TYPE NS#default#AGENTDEF List all resources of a type within a namespace (sorted by creation time)
GSI1SK S <creation_ms>#<name> 1706000000000#researcher Chronological sort within GSI1 partition (millis for uniqueness)
GSI2PK S NS#<ns>#AGENTDEF#<owner> NS#default#AGENTDEF#researcher Tasks and instances grouped by parent AgentDefinition
GSI2SK S TASK#<phase>#<ms> / INST#<phase>#<ms> INST#Ready#1706000003000 Filter by phase within parent group (begins_with query)
Data S protojson string {"metadata":{"name":"researcher"},...} Full proto resource as protojson — source of truth for deserialization
ResourceVersion S ULID 01JCXZ1234ABCDEF Optimistic concurrency token. Used in conditional writes and Watch resume.
FenceToken N UnixMilli 1740142800000 Leader election fencing. Writes rejected if stored FenceToken > write FenceToken.

Conditional Write Patterns
Create attribute_not_exists(PK) → ErrExists
Update ResourceVersion = :expected → ErrConflict
Delete attribute_exists(PK) → ErrNotFound
Fenced write FenceToken <= :token → ErrStaleFence
Lease acquire not_exists OR expired OR same_holder

ULID Resource Version
Format
01JCXZ 1234AB CDEF56 ←10ch→ ←————16ch random————→ Timestamp part (10 chars): milliseconds since epoch base32 Crockford encoded sortable lexicographically! Random part (16 chars): 80 bits of randomness monotonically increasing within same millisecond
Why ULIDs (not UUIDs)
Time-ordered Sortable in DynamoDB range key
Watch cursor Resume from any resource_version
No coordination Generated without distributed lock
Human-readable Timestamp visible in first 10 chars
Library github.com/oklog/ulid/v2
