02 — Control Plane
gRPC Watch-Driven Reconciliation

All control plane services connect to the api-server via gRPC Watch streams. When resources change, events are broadcast to all subscribers, which enqueue reconciliation work via deduplicated work queues.

⟳
AgentController
Watches AgentDefinitions. Compares desired replica count vs actual instance count. Creates or deletes instances to reconcile.
♥
HealthController
Watches AgentInstances. Evaluates heartbeat freshness, quality gates, and cost ceilings. Marks instances Failed when unhealthy.
⧖
TaskController
Watches AgentTasks. Handles timeouts, recovery retries, and stale task cleanup. Manages the task retry lifecycle.
Reconciliation Loop

Each controller follows the same pattern: WatchInformer receives events → keys enqueued to WorkQueue (deduplication, dirty tracking) → Controller.Reconcile(ctx, key) → observe desired/actual → compute delta → issue corrective writes → return Result (success, requeue, or requeue-after).

↻
WorkQueue
Dirty set + processing set. Same key not processed concurrently. Re-enqueue if modified while processing. Exponential backoff on error (5ms → 1000s).
⇄
Cross-Controller
Instance changes trigger both HealthController (re-evaluate) and AgentController (re-count). Informers feed multiple queues simultaneously.

04 — Scheduling
Multi-Objective Scheduling Pipeline

Tasks flow through a 5-phase scheduling pipeline that evaluates gang requirements, filters eligible instances, scores across multiple objectives, ranks candidates, and binds or preempts.

0
Gang Evaluation
Check nous.ai/gang-id label. If present, evaluate whether all gang members are available. States: GangNotApplicable → continue, GangWaiting → requeue 5s, GangTimeout → fail, GangReady → continue.
1
Filter
For each AgentInstance: phase must be Ready or Running, agentDefinition must match task.spec.agentRef, lastHeartbeat within timeout (30s default). Produces filtered candidate list.
2
Score
Each filtered instance scored by weighted plugins:
Cost
×0.3
Quality
×0.4
Latency
×0.3
Affinity
×0.3
total = Σ(plugin.score × weight) / Σ(weights)
3
Rank & Pick
Sort by total score descending → pick top candidate. Tie-breaker: instance with lowest current task load.
4
Bind or Preempt
Instance found: set task.status.assignedInstance, phase → Scheduled.
No instance + preemption: find lowest-priority running task (with anti-cascade guard), preempt it, re-enqueue original.
No instance + no preemption: increment unschedulableCount, requeue with backoff.

06 — Cognitive State
Checkpoint Lifecycle

Semantic checkpointing captures cognitive state at decision points during agent execution — reasoning depth, context utilization, intent, reasoning chain, tool history, and side effects. This enables recovery, preemption resume, and affinity-based scheduling.

decision_point
At each reasoning step / semantic boundary. Captures progressive context utilization.
periodic
Time-based snapshots (Phase 2). Safety net between decision points.
preemption
Emergency snapshot when task is preempted mid-flight. Enables resume on different instance.
completion
Final state after successful execution. Full cognitive context preserved.
failure
Snapshot for debugging post-failure. Captures state at point of failure.
Checkpoint Resume (Affinity)

New tasks with nous.ai/checkpoint-affinity: "01JR..3" get a +0.5 score boost for instances with matching checkpoint — enabling context-warm resume without cold reconstruction.

07 — HA
Leader Election & Split-Brain Protection

DynamoDB-based leader election with fence tokens prevents split-brain scenarios. Each leader write includes the fence token — if a stale leader revives, its writes are rejected due to token mismatch.
